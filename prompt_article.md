【地図ショートコードの挿入】
住所または施設名が判明している場合は、本文HTMLの詳細セクション末尾（参考情報セクションの直前）に以下のショートコードを挿入してください。

{{< map q="施設名または住所" >}}

qパラメータの記述ルール：
・URLエンコードは不要です。平文（日本語そのまま）で記述してください。
・「+」は使わず、スペースで区切ってください。
・施設名が一意に特定できる場合は施設名のみでかまいません。
・住所を含める場合は「施設名 住所」の順にスペース区切りで記述してください。

qパラメータの記述例：
  駅名のみ：             {{< map q="浦安駅" >}}
  施設名のみ：           {{< map q="バルドラール浦安アリーナ" >}}
  施設名＋住所：         {{< map q="生春巻きとタイ料理 エムズ 浦安市猫実3丁目24-1" >}}
  住所のみ：             {{< map q="千葉県浦安市港84番1" >}}

住所・施設名が不明な場合はショートコードを省略してください。

【記事化可否ごとの扱い】
■ そのまま記事化可
通常の記事として作成してください。
ただし、情報源の文章をそのまま転載せず、必ず言い換えてください。
■ 追加確認後に記事化
記事自体は作成して構いません。
ただし、不足情報や未確認部分を断定せず、本文では慎重な表現にしてください。
編集者メモに、掲載前に確認すべき点を明記してください。
■ 保留
原則として公開用の記事本文は作成しないでください。
代わりに、記事化を保留すべき理由、追加確認が必要な項目、確認先候補を編集者メモとして整理してください。
【既掲載・重複確認の扱い】
■ 未掲載と思われる
通常どおり記事を作成してください。
■ 掲載済みの可能性あり
記事は作成して構いませんが、編集者メモに「掲載済み確認推奨」と記載してください。
本文では既掲載かどうかに触れないでください。
■ 要サイト内確認
記事は作成して構いませんが、編集者メモに「サイト内検索で重複確認が必要」と記載してください。
本文では既掲載かどうかに触れないでください。
【4. アイキャッチ画像の設定】
記事のカテゴリー・内容に応じて、以下の優先順位でアイキャッチ画像の設定を決定してください。
決定結果は編集者メモの「アイキャッチ画像設定：」欄に出力してください。

■ 優先度1：個別写真がある場合
記事固有の写真素材（店舗外観・イベントビジュアルなど）を使用する場合は、以下を指定する。
画像ファイルの格納場所：static/images/posts/

編集者メモへの記載内容：
cover.image: /images/posts/ファイル名.jpg
cover.alt: '画像の内容を説明するテキスト'

■ 優先度2：開店・閉店カテゴリーで内容が明確な場合
カテゴリーが「開店・閉店」で、記事の内容が開店または閉店に明確に特定できる場合は、以下を指定する。
開店の記事 → cover.catimg: cat-open
閉店の記事 → cover.catimg: cat-close
開店・閉店どちらか不明、またはどちらも含む内容 → 指定しない（優先度3へ）

編集者メモへの記載内容（例）：
cover.catimg: cat-open

■ 優先度3：指定なし（カテゴリーデフォルトを自動適用）
優先度1・2のいずれにも該当しない場合は、cover フィールドの指定は不要。
カテゴリーに応じたデフォルト画像が自動的に使用される。

編集者メモへの記載内容：
カテゴリーデフォルト（指定不要）

カテゴリーとデフォルト画像の対応：
開店・閉店      → cat-open-close（開店・閉店どちらか不明の場合）
ニュース・事件事故 → cat-news
イベント        → cat-event
グルメ・カフェ   → cat-gourmet
子育て・教育     → cat-edu
スポーツ        → cat-sports
お知らせ        → cat-notice

【12. 品質チェック】
出力前に、以下を確認してください。
出力先頭にHTMLコメント2行があるか。
HTMLコメント2行が重複していないか。
HTMLコメントのタイトルと投稿タイトルが一致しているか。
公開日時はリサーチ対象日の翌日午前7時（JST）になっているか。
タイトルは40文字以内を目安に収まっているか。
エリア表記は正しいか。
店名・施設名は正式名称か。
本文HTMLに未確認情報を断定していないか。
本文HTMLに編集者向け指示が混ざっていないか。
本文HTMLにGoogleマップURLやWordPress作業指示が混ざっていないか。
本文HTML末尾に参考情報セクションが入っているか。
参考情報はMarkdownリンク形式になっているか。
著作権に配慮し、情報源の文章をそのまま使っていないか。
事件事故記事で個人情報や過度な詳細を書いていないか。
抜粋は100〜120文字程度か。
タグは3〜6個か。
カテゴリーは7種類の指定カテゴリーのいずれかになっているか（「ニュース・話題」等の類似名は不可）。
編集者メモにアイキャッチ画像設定が記載されているか。
開店・閉店記事で内容が明確な場合、cat-open / cat-close の指定があるか。
施設名または住所が判明している場合、本文HTMLに {{< map q="..." >}} ショートコードが含まれているか。
地図ショートコードのqパラメータがURLエンコードされていないか（ショートコードは平文で記述すること）。
編集者メモの地図欄に「本文HTMLへのショートコード挿入済み」または「住所未確認のため挿入不可」が記載されているか。
Hugo用フロントマターが出力されているか。
Hugo用フロントマターのYAML区切りが「---」（ハイフン3本）になっているか。
Hugo用フロントマターのdraftが「false」になっているか。
ファイル名にスペースや日付プレフィックス（20260526 等）が含まれていないか。
WordPress投稿に進めやすい形になっているか。
【13. 出力時の注意】
情報が不足している場合は、無理に補完しないでください。
不明点は本文HTMLで断定せず、編集者メモに追加確認事項として記載してください。
記事化可否が「保留」の場合は、公開用記事を無理に作成せず、保留理由と追加確認事項を整理してください。
参考情報の文章をそのまま転載しないでください。
地域メディアの記事を参考にする場合も、表現は必ず独自に言い換えてください。
読者にとって自然で、地域情報サイトらしい落ち着いた文体にしてください。
